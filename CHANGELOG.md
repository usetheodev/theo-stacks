# Changelog

## [Unreleased]

### Changed
- Hardened `.gitignore` for all 19 templates with comprehensive security rules: credentials (`*.pem`, `*.key`, `service-account*.json`), IDE files, OS files, coverage reports, and language-specific build artifacts. Environment variables now use `.env.*` with `!.env.example` safeguard

### Added
- API catch-all 404 route (`/api/[...not-found]`) to `node-nextjs`, `fullstack-nextjs`, and `monorepo-turbo` web app
- Middleware for API request tracking in `node-nextjs` and `fullstack-nextjs` templates
- Error boundary with styled retry button in `node-nextjs`, `fullstack-nextjs`, and `monorepo-turbo` web app
- Structured JSON logging (pino) to `node-nestjs` template via custom `PinoLogger` service
- Structured logging and graceful shutdown instrumentation to `monorepo-turbo` web app
- CORS headers to `monorepo-turbo` web app via `next.config.js`
- Graceful shutdown signal handlers (SIGTERM/SIGINT) to `ruby-sinatra` via `config.ru`
- Worker boot/shutdown logging to `ruby-sinatra` Puma config
- Template `php-slim`: PHP Slim Framework API server with CORS, JSON error handling, structured middleware
- Template `monorepo-rust`: Cargo Workspaces with Axum API + Tokio Worker + shared crate
- Template `monorepo-java`: Gradle multi-project with Spring Boot API + Scheduled Worker + shared package
- Template `monorepo-ruby`: Bundler multi-app with Sinatra API + WEBrick Worker + shared module
- Template `monorepo-php`: Composer multi-app with Slim API + CLI Worker + shared package
- PHP language support in CLI: CI workflow, database (Doctrine DBAL), Redis (Predis), Auth JWT (firebase/php-jwt), Auth OAuth (Guzzle), Queue (Symfony Messenger)
- `examples/` directory with 29 scaffolded projects (19 clean + 10 with addons) generated by the CLI
- Production-optimized Dockerfile for all 19 templates (multi-stage builds for Node, Go, Rust, Java)
- `.dockerignore` for all 19 templates with language-specific exclusions
- `GET /ready` readiness probe endpoint to all 19 templates (Kubernetes readiness probe)
- Example health check test for all 19 templates (Jest, Go testing, pytest, Rust tests, JUnit, Minitest, PHPUnit)
- Scaffold engine renames `dockerignore` â†’ `.dockerignore` during copy (same pattern as gitignore)
- `commands` section in `theo.yaml`: standardized wrappers for dev, build, test, lint, format, security across all 19 templates
- Per-app commands in monorepo `theo.yaml`: each app defines its own dev/build/test/lint commands
- Security audit commands per language: `npm audit`, `govulncheck`, `pip-audit`, `cargo audit`, `bundle audit`, `composer audit`, `./gradlew dependencyCheckAnalyze`

### Changed
- Improved `fullstack-nextjs` POST `/api/items` with input validation and JSON parse error handling
- Upgraded health endpoints in `node-nextjs`, `fullstack-nextjs`, and `monorepo-turbo` to include timestamp
- Updated `node-nextjs` and `fullstack-nextjs` not-found pages to use `next/link` instead of raw `<a>`
- Replaced NestJS built-in `Logger` with structured pino-based `PinoLogger` in `AllExceptionsFilter`
- Fixed `ruby-sinatra` Puma config: renamed `RAILS_MAX_THREADS` to `MAX_THREADS`

### Fixed
- `node-nestjs` logging was unstructured console output, now outputs JSON via pino
- `monorepo-turbo` web app missing error handling, CORS, logging, and 404 routes
- `ruby-sinatra` had no app-level signal handling for graceful shutdown

- Template category selection step: "What do you want to build?" with API/Backend, Frontend, Fullstack, Monorepo, Worker categories
- Template `node-worker`: background job processor with health endpoint for Kubernetes probes
- Field `type` on `TemplateInfo` to categorize templates (api, frontend, fullstack, monorepo, worker)
- Function `getTemplatesByType()` to filter templates by category
- Auto-skip template prompt when category has only one template
- PostgreSQL database layer with ORM auto-selection: Prisma (Node.js), GORM (Go), SQLAlchemy (Python)
- Flag `--database` (`-d`) to add database via CLI without prompt
- Database setup instructions in post-scaffold output
- Sample `User` model and connection boilerplate for each ORM
- CI workflow: tests on Node 18/20/22, template validation, security audit
- Release workflow: npm publish on version tag with provenance + post-publish smoke test
- Weekly validation workflow: scheduled Monday checks for template integrity and dependency vulnerabilities
